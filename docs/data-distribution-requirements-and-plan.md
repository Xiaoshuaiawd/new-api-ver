# 数据分布页面需求与落地方案（管理员）

## 1. 文档信息
- 文档名称：数据分布页面需求与落地方案
- 版本：V1.0
- 状态：可评审 / 可开发
- 适用范围：`new-api` 管理后台（Web 管理端）
- 时间口径：统一 `Asia/Shanghai`

## 2. 页面定位与菜单位置
### 2.1 页面定位
- 页面名称：`数据分布`
- 页面用途：面向管理员观察聊天日志存储规模、分布结构、token 结构和每日趋势

### 2.2 菜单位置（按你最新要求）
- 侧边栏一级菜单顺序必须为：`渠道管理 -> 数据分布 -> 订阅管理`
- 严禁将 `数据分布` 作为 `渠道管理` 的子级页面

### 2.3 权限与访问控制
- 仅管理员可见菜单
- 仅管理员可访问接口
- 非管理员访问页面路由或接口返回 `403`

## 3. 业务目标
1. 快速看到当前聊天日志存储规模。
2. 快速看到轮次分布、tools 使用分布、模型调用分布。
3. 快速看到 token 结构（`prompt_tokens` / `total_tokens`）分布。
4. 支持按日统计和跨天汇总查询。
5. 支持“聊天日志存储库”与“new 日志库（logs）”口径对比。
6. 与现有按天分表机制一致，确保查询与汇总正确。

## 4. 功能需求（FR）
### FR-01 总存储条数（P0）
- 展示当前时间范围内的总条数。
- 支持显示 `success`、`fail`、`all` 三种口径。

### FR-02 轮次分布饼图（P0）
- 固定轮次分桶：`1`、`2`、`3`、`4`、`5`、`10`、`15`、`20`、`其他`。
- 展示每个桶的条数与占比。

### FR-03 tools 调用条数（P0）
- 基于 `is_tools=1` 统计条数与占比。

### FR-04 模型调用分布（P0）
- 按 `model_name` 聚合调用次数。
- 页面展示图表 + 明细表（模型名、次数、占比）。

### FR-05 存储库对比（P0）
- 对比“聊天日志库”与“new 日志库（logs）”在同时间范围的核心指标。
- 最低要求对比：调用次数、prompt_tokens 总和、total_tokens 总和。
- 展示差值与差异比例。

### FR-06 按天分表存储（P0）
- 查询层按日期自动路由 `chat_log_success_YYYYMMDD` / `chat_log_fail_YYYYMMDD`。
- 缺失分表时不报错，按 0 处理。

### FR-07 prompt_tokens 分布饼图（P0）
- 分桶建议：
- `0`
- `1-100`
- `101-500`
- `501-1000`
- `1001-2000`
- `2001-4000`
- `4001+`
- 展示条数与占比。

### FR-08 total_tokens 分布饼图（P0）
- 分桶建议：
- `0`
- `1-200`
- `201-1000`
- `1001-2000`
- `2001-4000`
- `4001-8000`
- `8001+`
- 展示条数与占比。

### FR-09 每日统计列表（P0）
- 按天列出时间范围内每日指标：
- 日期
- 总条数
- 成功条数
- 失败条数
- tools 条数
- prompt_tokens 总和
- completion_tokens 总和
- total_tokens 总和
- 平均 `latency_ms`
- 当日 TOP1 模型

### FR-10 时间选择与汇总（P0）
- 默认时间：当天（上海时区）。
- 时间控件与“使用日志”页面一致（支持多天）。
- 跨天查询时，显示汇总 + 每日拆分。

## 5. 口径定义
### 5.1 时间口径
- 所有时间筛选与分表路由都使用 `Asia/Shanghai`。
- 当天定义为：`00:00:00` 到 `23:59:59.999`（上海时间）。

### 5.2 统计口径
- 调用条数：按日志行计数。
- tools 条数：`is_tools=1` 的日志行数。
- 模型调用次数：按 `model_name` 分组计数。
- 占比：`当前桶条数 / 口径总条数`，前端显示百分比。

### 5.3 对比口径（聊天日志库 vs logs）
- 时间范围一致。
- 若某指标维度不完全一致，页面展示“口径说明”提示。

## 6. 页面结构要求
1. 顶部筛选区：
- 时间范围（默认当天）
- 状态筛选（all/success/fail）
- 刷新按钮
2. 总览卡片区：
- 总条数
- tools 条数
- 成功/失败条数
- 对比差值摘要
3. 图表区：
- 轮次分布饼图
- prompt_tokens 分布饼图
- total_tokens 分布饼图
- 模型调用分布图
4. 列表区：
- 每日统计表
- 模型分布明细表（可与图联动）

## 7. 非功能需求（NFR）
1. 性能：
- 7 天范围查询建议 < 3s
- 30 天范围查询建议 < 8s
2. 稳定性：
- 分表缺失/空表不导致页面报错
3. 安全：
- 仅管理员接口与页面可见
4. 可观测性：
- 记录接口耗时、扫描分表数、缓存命中率（如启用缓存）

## 8. 验收标准（UAT）
1. 菜单位置正确：位于 `渠道管理` 与 `订阅管理` 之间。
2. 非管理员不可见且不可访问。
3. 默认当天统计准确。
4. 跨天汇总与每日统计一致。
5. 轮次 / token 分布饼图占比计算正确。
6. 模型分布与明细一致。
7. 库间对比能展示差值与比例。
8. 分表缺失场景页面可正常展示。

## 9. 落地方案

### 9.1 后端设计
#### 9.1.1 新增管理员接口
- `GET /api/admin/data-distribution`
- 入参：
- `start_time`（可选，默认当天起始）
- `end_time`（可选，默认当前时间）
- `status`（可选：`all|success|fail`，默认 `all`）

#### 9.1.2 返回结构（建议）
- `overview`：总览卡片数据
- `round_distribution`：轮次分布
- `tools_distribution`：tools 条数与占比
- `model_distribution`：模型分布
- `prompt_tokens_distribution`：prompt 分桶分布
- `total_tokens_distribution`：total 分桶分布
- `daily_stats`：每日统计列表
- `db_compare`：聊天日志库与 logs 对比
- `scan_tables`：本次扫描的分表列表（便于排障）

#### 9.1.3 查询实现
1. 按 `start_time/end_time` 生成上海时区日期列表。
2. 路由并扫描对应分表：
- `chat_log_success_YYYYMMDD`
- `chat_log_fail_YYYYMMDD`
3. 缺表跳过，累计聚合结果。
4. 同时间范围读取 `logs` 表聚合，构建 `db_compare`。

#### 9.1.4 数据分桶逻辑
- 在服务层统一实现分桶函数，前后端口径一致。
- 轮次桶和 token 桶固定配置，避免前端自行分桶导致偏差。

### 9.2 前端设计
#### 9.2.1 路由与权限
- 新增管理员路由：`/admin/data-distribution`
- 侧边栏插入位置：`渠道管理` 与 `订阅管理` 之间
- 菜单与路由均加管理员权限守卫

#### 9.2.2 页面组件
1. 时间筛选（复用使用日志时间选择器）
2. 总览卡片组件
3. 饼图组件（轮次、prompt_tokens、total_tokens）
4. 模型分布图 + 明细表
5. 每日统计表
6. 库间对比卡片/表格

### 9.3 性能与缓存
- 首版建议加 5 分钟查询缓存（key 含 `start/end/status`）。
- 大范围查询时返回扫描分表数量用于诊断。

### 9.4 监控与告警
- 监控项：
- 接口 QPS
- 平均响应耗时
- 慢查询数量
- 扫描分表数
- 告警项：
- 接口错误率异常
- 查询超时率异常

### 9.5 实施步骤
1. 后端：DTO、Service、Controller、Router 落地。
2. 后端：分表聚合与库间对比逻辑落地。
3. 前端：路由、菜单、页面布局、图表与表格落地。
4. 联调：口径核对（当天、多天、success/fail/all）。
5. 压测与优化：缓存、索引、慢查询处理。
6. UAT 验收并发布。

### 9.6 风险与规避
1. 风险：跨天多表查询慢  
规避：缓存 + 限制默认查询范围 + 异步优化。
2. 风险：两库统计口径差异  
规避：页面明确口径说明 + 指标定义固定。
3. 风险：缺表导致异常  
规避：缺表跳过策略 + 扫描分表结果可视化。
